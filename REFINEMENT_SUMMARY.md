## Refinement 1: Schema BNF Update (COMPLETE)

**File:** `schema.md`

**Changes:**
- **Broadened the `name` rule in BNF** to permit hyphens and dots in addition to underscores:
  ```
  Old: name ::= word ("_" word)*
  New: name ::= word ("_" word)* | word ("-" word)* | word ("." word)*
  ```
  This matches real-world usage (version numbers, domain qualifiers) seen in documented examples.

- **Clarified naming philosophy section** under Field 5 (Subject):
  - Underscores join words: `Strategic_Labelling_Framework`
  - Hyphens separate versions: `EGAP-v2.0.1`
  - Dots separate version components: `v2.0.1`
  - All conventions serve machine readability across time

- **Deduplicated epistemic marker table:**
  - Removed the second instance of `∴ — Derived by the framework (Rule D)`
  - Added explanatory note clarifying that `∴` is typically generated by framework inference (Rule D), not manually created by users
  - All four epistemic markers now appear exactly once: `∃`, `⊨`, `⊕`, `∴`

- **Updated example labels** to demonstrate version numbers and hyphens:
  ```
  ✓ !⊕∃F: EGAP-v2.0.1 ⊢ Tensor_Logic
  ✓ #∃F: Adaptive_Waypoint_System-v1.1
  ```

**Rationale:**
The original BNF was too restrictive. Real framework usage includes versioned identifiers and domain-qualified names. This refinement brings the formal grammar into alignment with documented practice without weakening the constraints on what constitutes a valid name.

## Refinement 2: Label Validator Tool (COMPLETE)

**Files:** `tools/validate_label.py`, `tools/README.md`

**Purpose:**
A lightweight, standalone reference validator that:
- Parses canonical labels against the schema
- Emits hard errors (label is invalid) and soft warnings (valid but flagged)
- Works as a CLI tool, importable Python module, and CI integration point
- Requires no external dependencies beyond Python 3.7+

**Key features:**
- **Correct epistemic/completeness parsing:** Understands that `∃` in `∃F`, `∃f`, `∃F+` is part of the completeness marker, while `⊨`, `⊕`, `∴` are explicit epistemic markers
- **Flexible object matching:** Validates that subjects and objects match the broadened `name` pattern (underscores, hyphens, dots)
- **Soft warning system:** Flags issues without rejecting labels (e.g., relation with no object, spaces in subject)
- **JSON output mode:** For machine consumption in CI/CD or API contexts

**Usage examples:**

CLI:
```bash
python3 validate_label.py "~∃F: Strategic_Labelling_Framework"
# Output: Status: VALID
```

Python:
```python
from validate_label import validate_label

result = validate_label("~∃F: My_Concept ⊢ Related")
if result.valid:
    print(f"Subject: {result.parsed['subject']}")
    print(f"Relations: {result.parsed['relations']}")
```

Integration in AI-assisted labelling:
```python
# AI generates a label
label = "~⊕∃F: New_Insight ⊢ Existing_Framework"

# Validate before returning to user
result = validate_label(label)
assert result.valid, f"Label generation failed: {result.errors}"
```

**Rationale:**
The framework had reached sufficient sophistication that maintaining consistency requires tooling. The validator ensures that:
1. Framework concepts remain machine-readable and parseable across sessions
2. AI systems generating labels can validate their own output
3. CI/CD can enforce label correctness before merging documentation
4. Human authors catch format drift early

## Changes to the Framework's Integrity

These refinements do **not** change the framework's core concepts:
- The five layers remain identical
- The ontological continuum (`∅ ∘ ∃ ∘ ∃f ∘ ∃F ∘ ∃F+ ∘ §`) is unchanged
- The four epistemic markers and their meanings are preserved
- The Curry-Howard extension and operational semantics are unaffected

What **has** changed:
- The grammar is now *honestly* formal (matches real practice)
- The framework can now *enforce* its own rules (via the validator)
- Integration with CI/CD and AI systems is now *documented and repeatable*

This is a refinement, not a redesign. The framework becomes easier to maintain and extend without sacrificing its original rigor.
